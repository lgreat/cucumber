UI Development

Place files in here that are being developed for mock-ups. Organize them however you like; nothing here gets deployed to the web server.

Keep in mind:

  1. make all pages XML (tags balanced, lower-case, etc.)
  2. you can develop pages within this hierarchy, creating local image, css and js directories if you want.
  3. Use "java-friendly" names for input fields. These start with a lowercase letter and use initial caps within. For example, "userEmailAddress" or "subscriptionNumber".
  4. use "id" on any significant element. Follow the "java-friendly" convention above. This will make java coding and testing easier.

Push to Site

Files that are ready to go should following these conventions:

  1. put images, javascripts and other required resources in the appropriate folders parallel to this directory. Therefore, use webapp/images and webapp/js and webapp/css.
  2. use the tag to point to the "webapp" folder
  3. make sure it looks right in this context.

Til is responsible for getting all the resources into the right folders outside of this folder.

Programmers will be responsible for moving the Html page. Be aware that this may involve breaking up the page into components.
