UI Development

Place files in here that are being developed for mock-ups. Organize them however you like; nothing here gets deployed to the web server.

Keep in mind:

  1. make all pages XML (tags balanced, lower-case, etc.)
  2. Til, you can develop pages within this hierarchy, creating local image, css and js directories if you want.
  3. Use "java-friendly" names for input fields. These start with a lowercase letter and use initial caps within. For example, "userEmailAddress" or "subscriptionNumber". These will probably get changed when we merge the file into Java.
  4. use "id" on any significant element. Follow the "java-friendly" convention above. This will make java coding and testing easier.

Push to Site

Files that are ready to go should following these conventions:

  1. put images, javascripts and other required resources in the appropriate folders inside the "webapp/res" folder, parallel to this directory. Therefore, use webapp/res/img and webapp/res/js and webapp/res/css.
  2. use the tag to point to the "webapp" folder to make sure your page works. See verisign1.htm to see it work.
  3. make sure it looks right in this context.
  4. tell the programmer it's ready to go.

Til is responsible for getting all the resources into the right folders outside of this folder. Programmers will be responsible for moving the Html page into the Java context. Be aware that this may involve breaking up the page into components.
